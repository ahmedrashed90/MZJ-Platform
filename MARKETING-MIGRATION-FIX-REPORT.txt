MZJ Marketing Migration Fix Report
Date: 2026-07-23
Source version: 1.18.2

ROOT CAUSE
- An older, now-excluded Local Publisher migration had already created marketing.publish_jobs with a different schema.
- The clean rebuild migration used CREATE TABLE IF NOT EXISTS for the canonical publish_jobs table.
- PostgreSQL kept the old table, so target_id was never created.
- The following index statement then failed with SQLSTATE 42703 because target_id did not exist.

SOURCE-LEVEL FIX
- Replaced the mixed additive approach with one canonical rebuild of the isolated marketing schema.
- Added a safety guard that aborts if real campaign, task, or photography-request business rows exist.
- Recreates marketing.campaigns, marketing.creatives, and marketing.tasks before extending the domain schema.
- Recreates marketing.publish_jobs with canonical target_id and publishing fields before any related indexes.
- Removes obsolete Checklist and Local Publisher tables/permissions from the canonical migration.
- Recreates the Operations photography-request foreign key by a stable named constraint.
- Added an automated migration structure checker to the normal typecheck chain.

VALIDATION PASSED
- Marketing canonical migration structure check.
- All Marketing index columns resolve against the canonical table definitions.
- All Marketing foreign-key target tables exist in the migration.
- Marketing full native rebuild structural check.
