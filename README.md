# MZJ Platform v1.9.2

منصة MZJ الموحدة المبنية بـReact/Vite على Vercel وPostgreSQL.

## CRM في هذا الإصدار

- سجل واحد دائم للعميل، مع طلب خدمة مستقل لكل رحلة كاش أو تمويل أو خدمة عملاء.
- عند وصول رسالة جديدة يبحث النظام عن العميل والطلب المفتوح قبل أي إنشاء أو توزيع.
- العميل الذي لديه طلب مفتوح يظل في نفس المحادثة ومع نفس المندوب.
- العميل الجديد أو صاحب الطلب المغلق تصله رسالة اختيار الخدمة، ثم يُنشأ الطلب ويُطبق التوزيع بعد الاختيار فقط.
- الأتمتة محصورة في دخول العميل واختيار الخدمة وإنشاء الطلب والتوزيع. لا توجد قواعد عامة أو شروط وإجراءات JSON.
- تغيير الحالات وإرسال الرسائل والقوالب ونقل العميل إجراءات يدوية.
- وكيل صندوق الوارد مستقل لمراقبة تأخر الرد البشري والتصعيد.
- سجل ملكية يوضح المسؤول السابق والجديد وسبب النقل والمنفذ والتاريخ.

## محادثة العميل

- رسالة العميل على اليسار.
- رد مستخدم CRM أو وكيل صندوق الوارد على اليمين.
- نفس الاتجاه يطبق على النصوص والصور والصوت والفيديو وPDF.
- نافذة العميل شاشة كاملة وتغلق بزر `Esc`.
- مربع الكتابة كبير ليستوعب القوالب الطويلة والمتغيرات.
- لا توجد قائمة «اختار قالب واتساب» داخل المحادثة.
- القالب يُحدد من ربط الحالة بالقالب، ويظهر في مربع الكتابة للمراجعة واستكمال المتغيرات قبل الإرسال اليدوي.

## إعدادات CRM

داخل الإعدادات > إعدادات CRM:

- دخول وتوزيع العملاء.
- الحالات.
- بيانات العميل.
- المصادر.
- القوالب والرسائل.
- ربط الحالة بالقالب.
- قواعد التوزيع.
- ربط المنصات والـWorkers.

إعدادات كل قناة تحفظ في سجل القناة داخل تبويب ربط المنصات، ولا توضع داخل شاشة المحادثة.

## واتساب / مرسال

المنصة تعتمد وركر واتساب/مرسال الفعلي الذي يستقبل على:

- `POST /webhook/mersal`

ويستقبل طلبات الإرسال من CRM على مسار واحد:

- `POST /send/mersal`

الـWorker يحدد داخليًا نوع الإرسال:

- النص الحر إلى `/api/wpbox/sendmessage`.
- القالب إلى `/api/wpbox/sendtemplatemessage` عند وجود `template_name`.

في إعدادات المنصة اجعل «مسار إرسال النص والقوالب»:

`https://YOUR-WORKER/send/mersal`

واضبط مسار الوسائط بشكل مستقل حسب النسخة المنشورة من Worker.

## المهام المؤجلة

لا يستخدم المشروع Vercel Cron. وكيل صندوق الوارد يستخدم Cloudflare Queue من خلال Worker المجدول الموجود في:

- `automation-scheduler-worker/`
- `workers/MZJ-Automation-Scheduler-Worker-v1.0.0-FULL.txt`

متغيرات Vercel:

- `DATABASE_URL`
- `MZJ_SETUP_KEY`
- `MZJ_GATEWAY_SECRET`
- `AUTOMATION_SCHEDULER_URL`
- `AUTOMATION_SCHEDULER_SECRET`

متغيرات Worker المجدول:

- `PLATFORM_AUTOMATION_CALLBACK_URL`
- `AUTOMATION_SCHEDULER_SECRET`
- Queue binding باسم `AUTOMATION_QUEUE`
