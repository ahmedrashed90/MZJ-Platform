# ربط إلغاء طلبات البيع من NEXT ERP — v1.19.4

## إعداد ERPNext

يظل Webhook الاعتماد الحالي `on_submit` كما هو. يضاف Webhook واحد للإلغاء على نفس الرابط والمفتاح:

- DocType: `Sales Order`
- Doc Event: `on_cancel`
- Request Method: `POST`
- Request URL: `/api/integrations/erpnext/sales-order`
- Header: `X-MZJ-ERPNext-Key`
- JSON: `integration-assets/MZJ-ERPNext-Sales-Order-Cancel-Webhook-JSON.txt`

حقل `creation` جزء من هوية نسخة الطلب، لذلك يمكن استقبال طلب جديد بنفس رقم التسلسل بعد إلغاء النسخة السابقة دون اعتباره مكررًا.

## السلوك

- المرحلة التي تنفذ على سيارة تطبق على جميع سيارات نفس طلب التتبع.
- نجاح SMS+ يجعل زر المرحلة أخضر ويظل كذلك بعد إعادة فتح الصفحة.
- إلغاء الطلب يحفظه كسجل ملغي ويوقف المراحل وSMS+.
- السيارة تحت التسليم ترجع إلى متاح للبيع فقط إذا لم يوجد طلب بيع أحدث ولم تكن مسلمة أو مؤرشفة.
- دورة الموافقات النشطة تغلق كسجل ملغي.
- تبويب طلب البيع داخل السيارة يحتفظ بالطلب الملغي ويعرض الطلب النشط إن وجد.
- صلاحيات الواجهة الحالية لا تتغير.
