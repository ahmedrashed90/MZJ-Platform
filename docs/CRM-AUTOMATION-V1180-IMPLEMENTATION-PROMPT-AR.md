# البرومت المرجعي المعتمد لأي تعديل لاحق على أوتوميشن MZJ v1.18.0

اعمل على آخر سورس معتمد للمنصة فقط، ولا تنشئ باتشًا فوق باتش أو مسارًا موازيًا أو دوال Fix مؤقتة.

نظام إعدادات الأوتوميشن الموجود في `الإعدادات → إعدادات CRM → إعدادات الأوتوميشن` هو المصدر الوحيد لتعريف رسائل البداية والاختيارات والردود المقبولة والخطوات والتحقق وسياسات التشغيل والإجراء النهائي والمنصات والـWorkers.

محرك `دخول وتوزيع العملاء` يظل المصدر الوحيد لقواعد الأقسام والفروع والموظفين وRound Robin وتوزيع المبيعات وخدمة العملاء والكول سنتر. ممنوع نسخ قواعد التوزيع إلى محرك الأوتوميشن أو الـWorker.

Facebook Worker ناقل تقني فقط. يحافظ على الاستقبال والإرسال والوسائط وPSID وRoutes الحالية ونتيجة المزود الحقيقية، ولا يحتوي على أي رسالة أو سؤال أو قرار خدمة أو إنشاء عميل أو توزيع.

أي تعديل جديد يجب أن:

1. يغير تعريفات الفلو المركزية أو الخدمات المركزية الموجودة، لا أن يضيف تنفيذًا موازيًا.
2. يحافظ على Idempotency للأحداث والإجابات والرسائل والإجراء النهائي.
3. يمنع أكثر من جلسة نشطة لنفس العميل أو المحادثة.
4. لا ينتقل للخطوة التالية إلا بعد نجاح إرسال السؤال الحالي من المزود.
5. يحتفظ بالإجابة الصحيحة عند فشل إرسال الخطوة التالية ويتيح إعادة محاولة آمنة.
6. يطبق التعديل على الواجهة والـAPI وقاعدة البيانات والصلاحيات عند الحاجة.
7. يستخدم Migration واضحة، ولا يعدل Schema وقت تشغيل الـAPI.
8. لا يغير أي صفحة أو منطق خارج نطاق الطلب.
9. يضيف أو يحدث اختبارات قبول تغطي السيناريو الجديد والتكرار والفشل.
10. يسلّم السورس كاملًا والـMigration والـWorker عند تعديله وتقرير الملفات والاختبارات.

الفلو الافتراضي الحالي الذي يجب الحفاظ عليه ما لم يطلب تعديل صريح:

- رسالة ترحيب واختيار كاش أو تمويل أو خدمة عملاء.
- الكاش: إنشاء/تصنيف مركزي ثم توزيع مركزي ثم رسالة النهاية.
- التمويل: الاسم ثم السيارة ثم رقم الجوال والتحقق ثم إنشاء/تصنيف وتوزيع مندوب التمويل والكول سنتر ثم رسالة النهاية.
- خدمة العملاء: إنشاء/تصنيف مركزي ثم توزيع مركزي ثم رسالة النهاية.
