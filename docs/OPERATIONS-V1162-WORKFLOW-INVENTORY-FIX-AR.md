# تعديلات العمليات v1.16.2

تم تنفيذ هذه النسخة مباشرة على مصدر v1.16.1 بدون طبقة Patch مستقلة.

## نطاق التعديل

- نوافذ الداش بورد الخاصة بالمستودع والوكالة والصالة والقادسية والملتقى ونواقص السيارات.
- جدول مخزون السيارات وملف السيارة.
- صفحة الحركة وصفحة الموافقات.
- API العمليات ومخطط PostgreSQL الخاص بالتشييك والموافقات.

## إصلاح حركة الوكالة

تم توحيد كتابة سجل التشييك مع مخطط قاعدة البيانات، وإضافة ترقية آمنة للنسخ القديمة من جدول `operations.vehicle_check_history` حتى يتوفر عمود `note` قبل أول عملية حركة.

## التسليم النهائي

- عند اختيار `delivered` والموافقتان مكتملتان، تنفذ الحركة مباشرة.
- عند نقص موافقة، تُحفظ الحركة المطلوبة داخل سجل الموافقات بدون تغيير مكان أو حالة السيارة.
- عند اكتمال الموافقة الثانية، تنفذ حركة التسليم النهائية تلقائيًا بنفس بيانات الطلب ومنفذ الحركة الأصلي.

## المسح النهائي

يتطلب المسح سببًا وكتابة رقم الهيكل مطابقًا. وتُحذف السجلات المرتبطة بالسيارة في التشييك والموافقات والحركات وطلبات النقل والتصوير وروابط التراكينج داخل Transaction واحدة، مع إبقاء سجل تدقيق مستقل لعملية المسح نفسها.
