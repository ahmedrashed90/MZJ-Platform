# إعادة بناء نظام التسويق Native داخل منصة MZJ — v1.19.0

## الهدف

تم استبدال صفحة التسويق الفارغة بموديول تسويق كامل مبني داخل المنصة نفسها باستخدام React وTypeScript وواجهات API الحالية وPostgreSQL، بدون نسخ صفحات قديمة أو إضافة باتشات متراكمة، وبدون تغيير منطق CRM أو العمليات أو التراكينج.

## الفلو المنفذ

1. **إنشاء الحملة من صفحة كاملة بخمس مراحل**
   - بيانات الحملة والهدف والـBrief والتواريخ.
   - الكرييتيفات وربط كل نسخة بالأقسام والمستخدمين والسيارات.
   - ميزانية مستقلة لكل نسخة كرييتيف.
   - جدول النشر والمنصات وأنواع النشر.
   - مراجعة نهائية ثم الحفظ.

2. **نسخ كرييتيف مستقلة**
   - الكمية تنشئ سجلات مستقلة مثل `REEL-01` و`REEL-02`.
   - لكل نسخة سياراتها وألوانها وميزانيتها وتاسكاتها وجدول نشرها ومسارات RAW/OUTPUT الخاصة بها.
   - تم منع تكرار `instanceKey` حتى بعد حذف نسخة وإضافة نسخة جديدة.

3. **اعتماد الهيكل**
   - الحالة الأولى: `في انتظار اعتماد الهيكل`.
   - بعد الاعتماد يتم إنشاء Task Template لكاتب المحتوى وتكليفات تنفيذ مستقلة للتصميم أو المونتاج أو التصوير.
   - لا يمكن تعديل بنية الحملة بعد اعتماد الهيكل، بما في ذلك الأدمن، حفاظًا على سلامة السجلات.

4. **Task Template**
   - الاسم المقترح.
   - الرسالة الأساسية.
   - السكريبت الأساسي.
   - Hook.
   - CTA.
   - حفظ، إرسال للاعتماد، اعتماد، أو إعادة للتعديل.

5. **فتح التنفيذ بعد اعتماد المحتوى**
   - تاسكات التصميم/المونتاج/التصوير تبدأ بحالة `في الانتظار` وبدون موعد تنفيذ.
   - بعد اعتماد Task Template تتحول إلى `جاهز للتنفيذ` فقط.
   - موعد التنفيذ لا يمكن تحديده إلا بعد اعتماد المحتوى.
   - المنفذ يضيف خطوات التنفيذ ويرفع مسار أو رابط الملف النهائي.
   - الأدمن يعتمد الاستلام أو يعيد التاسك للتعديل.

6. **الاستلام وتجهيز النشر**
   - لا تنتقل الحملة إلى تجهيز النشر قبل اعتماد كل Task Template واستلام كل تكليفات التنفيذ.
   - الحملات التي لا تحتوي عناصر نشر تنتهي كحملات أوفلاين بعد اكتمال التاسكات.
   - الحملات الرقمية تنتقل إلى `تجهيز النشر` ثم `مجدولة` ثم `مكتملة` حسب حالة كل عناصر النشر.

7. **الأجندة والتقويم**
   - إضافة وتعديل وحذف مواعيد التسويق والاجتماعات والتسليمات.
   - تقويم شهري موحد يعرض الأجندة ومواعيد النشر.

8. **إعدادات التسويق**
   - إدارة أنواع الكرييتيف وربطها بالأقسام.
   - إدارة المنصات وأنواع النشر وحالة الربط.
   - TikTok مضبوط على `Sandbox / Draft Upload`.
   - Snapchat مضبوط على `بانتظار موافقة Public Profile API`.

## ربط الأقسام الافتراضي

- تصوير المعرض: المحتوى + التصوير.
- REEL / VIDEO / STORY: المحتوى + المونتاج.
- POST / CAROUSEL / PANNER / MOTION / GIF / PRINT / MZJ-INTERIAL: المحتوى + التصميم.

## الصلاحيات

تمت إضافة صلاحيات مستقلة لعرض التسويق، إدارة الحملات، اعتماد الهيكل، تنفيذ واعتماد Task Template، تنفيذ واعتماد التكليفات، تجهيز النشر، وإدارة الإعدادات.

المستخدم العادي يرى التاسكات المسندة إليه فقط. الربط بكاتب المحتوى يُحفظ كمرجع داخل تاسك التنفيذ، لكنه لا يمنح كاتب المحتوى حق تنفيذ تاسك المصمم أو المونتير أو المصور.

## قاعدة البيانات

تمت إضافة طبقة Schema ذاتية التشغيل `ensureMarketingSchema()` لضمان تهيئة الجداول والحقول والصلاحيات بصورة idempotent عند تشغيل API التسويق على قاعدة موجودة، بالإضافة إلى Migration مستقلة للنشر اليدوي.

الجداول الجديدة أو الموسعة تشمل:

- `marketing.campaigns`
- `marketing.creatives`
- `marketing.tasks`
- `marketing.agenda_items`
- `marketing.publishing_items`
- `marketing.creative_type_settings`
- `marketing.platform_settings`
- `marketing.activity_log`

## مسارات RAW وOUTPUT

يتم توليد وحفظ مسار الحملة ومساري `01-RAW` و`02-OUTPUT` لكل نسخة كرييتيف على هيكل قرص Z. زر التجهيز يحفظ تأكيد المسارات داخل قاعدة البيانات. الإنشاء الفيزيائي على قرص Z يحتاج الجسر المحلي/الـPublisher الموجود على أجهزة العمل؛ هذا الجسر غير موجود في سورس المنصة المرفوع، لذلك لم تتم إضافة عنوان أو بروتوكول تخميني قد يعطل النظام.

## النشر

1. نشر السورس.
2. ستعمل طبقة `ensureMarketingSchema()` تلقائيًا عند أول طلب إلى API التسويق.
3. بديلًا عن ذلك، يمكن تشغيل:
   - `database/migrations/20260723_marketing_native_rebuild_v1190.sql`
4. إعادة تسجيل دخول مستخدمي التسويق بعد النشر تضمن تحميل الصلاحيات الجديدة في الجلسة.

## النتيجة

أصبح `/marketing` نظامًا فعليًا داخل المنصة، وليس Placeholder، مع صفحات الحملات والتاسكات والأجندة وتجهيز النشر والتقويم والإعدادات، وبفلو حالات واعتمادات موحد ومقفل على الترتيب المعتمد.
