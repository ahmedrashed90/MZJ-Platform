# تقرير المستخدمين والصلاحيات النهائية

## حالة المستخدمين الحالية

لم يُرفق جدول مستخدمين إنتاجي نهائي. طبقًا للمعلومات المتاحة توجد أربعة حسابات اختبار فقط. لذلك لم تُنشأ أسماء أو إيميلات أو صلاحيات إنتاجية مفترضة.

## سلوك الترحيل

- المستخدم الموجود يُطابق بواسطة User ID داخلي أثناء الترحيل، مع بقاء البريد مرجعًا فريدًا للإدارة والاستيراد.
- يحتفظ المستخدم بكلمة المرور وUser ID والسجلات التشغيلية.
- تُرحل الأدوار والفروع والأقسام الحالية إلى نطاقات الأنظمة المناسبة.
- مدير النظام الحالي يحصل على قالب الإدارة المركزية.
- المستخدمون الآخرون يحصلون على أقل نطاق متوافق مع أدوارهم وبياناتهم الحالية، ثم يمكن تعديلهم من الإعدادات.

## الإمكانات المتاحة من الواجهة

- إنشاء وتعديل وتعطيل وإعادة تفعيل المستخدم.
- اختيار أكثر من دور.
- تفعيل أو تعطيل كل نظام دون حذف إعداداته.
- تعيين دور مستقل داخل كل نظام.
- تحديد نطاق البيانات والفروع والأقسام لكل نظام.
- مشاهدة الصلاحيات الموروثة.
- إضافة سماح فردي أو منع صلاحية موروثة.
- معاينة الصلاحيات النهائية.
- نسخ إعدادات الوصول من مستخدم إلى نموذج مستخدم آخر بعد المراجعة.
- إعادة ضبط الاستثناءات الفردية إلى قوالب الأدوار.
- اختيار الفرع الأساسي والقسم الأساسي صراحة لكل نظام.
- البحث والفلترة ومشاهدة منفذ آخر تعديل في السجلات.

## النقل التنظيمي

نقل المستخدم إلى فرع أو قسم آخر يعدل نطاقه فقط. لا ينقل تلقائيًا العملاء أو الطلبات أو التاسكات المسندة، ولا يغير أي فلو تشغيلي.

## قالب الاستيراد
