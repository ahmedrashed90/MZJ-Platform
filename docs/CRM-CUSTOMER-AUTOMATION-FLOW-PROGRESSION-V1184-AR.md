# تقرير إصلاح أوتوميشن العملاء v1.18.4

## المطلوب المنفذ

- إعادة إظهار سياسة تشغيل الأوتوميشن داخل إعدادات CRM:
  - مع كل رسالة واردة خارج فلو نشط.
  - مرة كل 24 ساعة.
  - مدة مخصصة بالدقائق أو الساعات أو الأيام.
- ربط السياسة بالحفظ وبمحرك التشغيل المركزي نفسه.
- إصلاح توقف فلو التمويل بعد سؤال الاسم.

## إصلاح فلو التمويل

أصبح الانتقال ثابتًا وفعليًا بالشكل التالي:

1. رسالة بداية التمويل + سؤال الاسم.
2. بعد حفظ الاسم تُحفظ حالة الفلو مباشرة على خطوة السيارة، ثم يُرسل سؤال السيارة.
3. بعد حفظ السيارة تُحفظ حالة الفلو مباشرة على خطوة رقم الجوال، ثم يُرسل سؤال رقم الجوال.
4. بعد رقم الجوال تُرسل رسالة النهاية ويُغلق الفلو كمكتمل.

تم إلغاء الاعتماد على حالة انتقالية مؤقتة بين حفظ الإجابة وتحديد السؤال التالي. كما أضيف استرجاع آمن لأي تشغيل قديم عالق وكانت الخطوة التالية محفوظة بالفعل.

## حدود التعديل

- لم يتم تغيير الخدمات الثلاث أو ترتيبها.
- لم يتم تغيير الأقسام أو الفروع أو حقول حفظ إجابات التمويل.
- لم يتم تغيير منطق توزيع الموظفين.
- لم يتم تعديل وركر Facebook؛ الوركر الحالي متوافق لأن انتقال الخطوات يتم داخل المنصة.
- لا توجد Migration جديدة مطلوبة؛ أعمدة سياسة التشغيل موجودة بالفعل في بنية إعدادات الأوتوميشن.
