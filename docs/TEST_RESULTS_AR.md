# نتائج الاختبارات

## اختبارات آلية منفذة

- فحص Imports ومسارات API.
- فحص بنية CRM وعدم وجود Sample Data.
- فحص حساب اكتمال العميل.
- فحوص مرجع CRM وتدفق غير المقروء وحالات التمويل.
- فحوص إعدادات CRM ومزامنة قوالب مرسال.
- فحوص توزيع الدخول وWorker الإرسال الموحد.
- فحص TypeScript الكامل `tsc -b`.
- فحص مركزي جديد `scripts/check-central-access-control.mjs` (64 تحققًا).
- Build إنتاجي بواسطة Vite.

## ما يتحقق منه فحص الصلاحيات المركزي

- وجود مفاتيح إعدادات المستخدمين المنفصلة.
- وجود التبويبات الستة المطلوبة.
- وجود تبويبات الأنظمة الأربعة داخل ملف المستخدم.
- عدم حذف صلاحيات النظام عند تعطيله.
- أولوية المنع الفردي.
- فلترة صلاحيات النظام المعطل.
- إبطال Cache/Version الصلاحيات.
- تحويل المسارات القديمة إلى الإعدادات المركزية.
- عدم استخدام دور `admin` كبديل عن صلاحية داخل API المركزي.
- حماية Endpoints CRM بمفاتيح الصفحة والإجراء المناسبة.
- منع الداش بورد الموحد من استعلام أنظمة غير مسموحة وتطبيق Scope CRM داخل SQL.
- إخفاء أقسام الداش بورد غير المسموحة من الواجهة.
- حماية API بيانات الفروع والأقسام والأدوار بصلاحيات إعدادات صريحة.
- وجود Migration وSeed.

## اختبارات تشغيلية يجب تنفيذها على بيئة Staging

تحتاج قاعدة بيانات فعلية ومستخدمين حقيقيين، وتشمل:

1. مستخدم دون نظام يحاول فتح الرابط واستدعاء API ويحصل على 403.
2. مستخدم يرى النظام ولا يرى صفحة محددة.
3. مستخدم يرى الصفحة ولا يستطيع الحذف أو النقل أو التصدير.
4. تعطيل CRM ثم إعادة تفعيله والتأكد من عودة الدور والاستثناءات السابقة.
5. مندوب مبيعات يرى العملاء المسندين إليه فقط.
6. مندوب كول سنتر يرى العملاء المرتبطين به فقط.
7. مدير فرع يرى الفروع والأقسام المختارة فقط.
8. منع فردي يتغلب على صلاحية الدور.
9. محاولة منح صلاحية أعلى من صلاحيات المنفذ تُرفض.
10. تعطيل مستخدم يبطل جلسته القديمة فورًا.
11. مراجعة سجل تغييرات الصلاحيات وسجل النشاط الأمني.
12. تشغيل Migration وSeed مرتين والتأكد من عدم التكرار.
13. التحقق أن الداش بورد الموحد لا يعيد أرقام CRM أو أي نظام معطل للمستخدم.
14. التحقق أن مستخدم CRM محدود النطاق يرى في الداش بورد نفس العدادات الواقعة داخل نطاقه فقط.

## نتيجة الاختبارات المحلية

تسجل نتيجة الأمرين التاليين في ملف التسليم:

```bash
npm run typecheck
npm run build
```

## النتيجة النهائية

- `npm run typecheck`: ناجح.
- `npm run build`: ناجح.
- Vite production build: ناجح، وتم توليد ملفات الإنتاج دون أخطاء.
