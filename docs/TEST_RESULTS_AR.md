## ما تغطيه الاختبارات المركزية

- المصدر الوحيد داخل الإعدادات المركزية.
- تفعيل وتعطيل الأنظمة مع الاحتفاظ بالإعدادات المخزنة.
- صلاحيات النظام والصفحة والإجراء ومراحل سير العمل.
- ترتيب المنع الفردي ثم السماح الفردي ثم الدور ثم المنع الافتراضي.
- منع تعديل الحساب أو الدور المؤثر على المستخدم من نفس الجلسة.
- منع منح صلاحيات أو نطاقات أعلى من صلاحيات المنفذ.
- فصل `settings.roles.manage` عن `settings.permissions.manage`.
- إبطال الجلسات عند تغيير الدور أو النطاق أو حالة المستخدم.
- حماية CRM والتسويق والعمليات والتراكينج من الواجهة والـAPI والسيرفر.
- تطبيق Data Scope داخل استعلامات السيرفر.
- Tracking Token العام بدل VIN أو رقم الطلب.
- مفاتيح مستقلة لحذف وإلغاء وإنهاء طلبات العمليات وتعديل VIN.
- عدم وجود شروط صلاحيات بإيميلات ثابتة أو `localStorage` أو تجاوز `canTransfer=true`.

## اختبارات الحفاظ على الفلو

تم تشغيل Scripts التحقق الموجودة في السورس للـCRM والأوتوميشن والتقارير والعملاء اليدويين ومرسال وFacebook والـERPNext والعمليات والموافقات والأرشفة والتراكينج والـWorkers. جميعها نجحت بعد تحديث Assertions التي كانت تبحث عن صفحات أو عبارات صلاحيات قديمة ألغيت لصالح النظام المركزي.

لم تتغير مراحل أو حالات أو ترتيب فلو أو معادلات تشغيلية نتيجة تحديث الاختبارات؛ التحديثات تقيس بوابات الصلاحيات الجديدة حول نفس الفلو.

## قيود بيئة الفحص

لم يتم تنفيذ Build إنتاجي كامل بواسطة `pnpm run build` لأن `pnpm` والحزم المحلية غير متاحة في بيئة العمل، ومحاولة Corepack احتاجت اتصالًا خارجيًا غير متاح. لذلك لا يُذكر Build إنتاجي كاختبار ناجح.

تم بدلًا من ذلك تنفيذ:

1. Parse كامل لكل ملفات TypeScript وTSX باستخدام TypeScript المتاح في البيئة.
2. جميع Scripts التحقق الثابتة الموجودة بالمشروع.
3. فحص Imports الخاص ببناء Vercel.
4. فحص مركزي مستقل للصلاحيات والـMigrations والـSeed والحماية.

بعد تنزيل الحزم في بيئة النشر يجب تشغيل:

```bash
pnpm install --frozen-lockfile
pnpm run typecheck
pnpm run build
```
