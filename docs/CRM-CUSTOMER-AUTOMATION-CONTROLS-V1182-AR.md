# تقرير تعديل إعدادات الأوتوميشن وفلو التمويل v1.18.2

## المطلوب المنفذ

- إعادة إعداد **تشغيل/إيقاف الأوتوميشن** داخل إعدادات CRM.
- إعادة حقل **اسم الأوتوميشن**.
- إعادة قسم **المنصات والـWorkers** من المصدر الفعلي `crm.integration_endpoints`.
- السماح بتشغيل أو إيقاف الأوتوميشن لكل منصة واختيار الـWorker التابع لها.
- منع حفظ ربط Worker غير تابع للمنصة أو تشغيل Worker غير نشط.
- الحفاظ على الفلو الثلاثي الثابت بدون السماح بتعديل الأقسام أو الفروع أو الحقول أو ترتيب الخطوات.

## فلو التمويل النهائي

1. اختيار مبيعات التمويل.
2. إرسال رسالة بداية التمويل مع سؤال الاسم.
3. بعد حفظ الاسم: إرسال سؤال السيارة.
4. بعد حفظ السيارة: إرسال سؤال رقم الجوال.
5. بعد التحقق من رقم الجوال وحفظه: إرسال رسالة نهاية التمويل.

تم جعل الانتقال يعتمد على كود الخطوة التي تمت الإجابة عنها، وليس على رقم فهرس قابل للانحراف. كما يتم تثبيت حالة الخطوة التالية في قاعدة البيانات قبل وضع الرسالة في طابور الإرسال، لمنع توقف الفلو في الحالة المؤقتة `classifying`.

## ما لم يتغير

- منطق توزيع الموظفين.
- الأقسام والفروع المرتبطة بالخدمات الثلاث.
- ترتيب الخدمات.
- حقول التمويل: الاسم، السيارة، رقم الجوال.
- صفحات CRM الأخرى.
- العمليات والتراكينج والتسويق.
- وركر Facebook؛ النسخة الحالية تنقل الرسائل والأزرار للمنصة ولا تدير خطوات الفلو، لذلك لا تحتاج تعديلًا إضافيًا لهذا الإصلاح.

## الفحوصات

- نجاح 22 فحصًا قائمًا للمشروع.
- نجاح فحص الأوتوميشن المحدث لفلو: الاسم → السيارة → رقم الجوال → النهاية.
- نجاح Transpile للملفات TypeScript وTSX المعدلة.
- لم يتم تشغيل Build إنتاجي كامل لعدم وجود جميع حزم المشروع في بيئة العمل.
