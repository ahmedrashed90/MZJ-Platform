# إصلاح تسجيل دخول مدير النظام - v1.19.2

## السبب الجذري

بعد نشر طبقة الصلاحيات المركزية على قاعدة بيانات منصة موجودة مسبقًا، كان مسار تسجيل الدخول يستخدم أعمدة وجداول الإصدار الجديد مثل `permission_version` و`core.user_systems` قبل ضمان تطبيق Migration الصلاحيات على قاعدة الإنتاج. كلمة المرور كانت صحيحة، لكن إنشاء الجلسة أو تحميل الصلاحيات كان يفشل، فتظهر رسالة «تعذر تسجيل الدخول».

## الحل البنيوي

- إنشاء خدمة واحدة `ensureAccessControlSchema()` داخل ملف مخطط الصلاحيات المركزي.
- فحص خفيف لحالة المخطط قبل المصادقة.
- تشغيل Migration المركزية الكاملة فقط عند الحاجة.
- استخدام PostgreSQL advisory lock لمنع تنفيذ DDL متزامن من أكثر من Serverless instance.
- استدعاء الخدمة من تسجيل الدخول، إنشاء الجلسة، قراءة الجلسة، وتحميل ملف المستخدم.
- توحيد تهيئة المنصة على نفس الخدمة بدل وجود مسارين مختلفين.
- إضافة حالة إصدار `core.access_control_schema_state` لضمان أن قاعدة البيانات وصلت إلى الإصدار المطلوب.

## ما لم يتغير

لم تتغير كلمة مرور المستخدم، User ID، الأدوار، الفروع، الأقسام، الصلاحيات المخزنة، أو أي فلو تشغيلي في الأنظمة الأربعة.
