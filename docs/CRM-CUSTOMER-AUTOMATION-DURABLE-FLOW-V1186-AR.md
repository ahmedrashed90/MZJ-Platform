# إصلاح فلو تمويل الأوتوميشن — v1.18.6

## المشكلة المؤكدة

رسالة العميل الواردة كانت تدخل المحادثة، لكن انتقال الفلو بعد إجابة الاسم لم يكن مضمونًا. عند فشل إرسال سؤال السيارة لأي سبب، كان حدث الرسالة ينتهي دون مسار إعادة محاولة مضمون، وقد يظل التشغيل في حالة انتقالية لا يعرف منها إن كانت الإجابة حُفظت أم السؤال التالي أُرسل.

## التنفيذ

تم إعادة تنظيم انتقال فلو التمويل داخل محرك الأوتوميشن المركزي ليصبح مسارًا ثابتًا ودائمًا:

1. الاسم.
2. السيارة.
3. رقم الجوال.
4. رسالة النهاية.

كل انتقال يحفظ الرسالة التالية كـ Pending Delivery داخل نفس سجل تشغيل الأوتوميشن قبل محاولة إرسالها. لا ينتقل التشغيل إلى انتظار إجابة الخطوة التالية إلا بعد نجاح إرسال سؤالها.

إذا فشل الإرسال:

- تظل الرسالة المطلوبة محفوظة في سجل التشغيل.
- لا تضيع إجابة العميل السابقة.
- لا يتم اعتبار السؤال التالي مرسلًا.
- تتم إعادة محاولة نفس رسالة الإرسال بنفس Idempotency Key دون إنشاء رسالة مكررة.
- يعاد فتح حدث الأوتوميشن الفاشل عند إعادة وصول Webhook بدل إغلاقه نهائيًا.
- يعيد Endpoint الاستقبال حالة Retryable للوركر حتى يعيد Meta تسليم الحدث.

## التوافق مع التشغيل القديم

تمت إضافة استرجاع للحالات القديمة العالقة في `classifying`. عند وصول حدث جديد بعد مرور مدة الأمان، يتم تحديد السؤال الذي كان يجب إرساله وإرساله أولًا بدل استهلاك رسالة العميل على خطوة غير ظاهرة له.
