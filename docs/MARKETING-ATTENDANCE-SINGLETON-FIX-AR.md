# إصلاح إعدادات حضور التسويق — v1.19.3

## السبب الجذري
كان جدول `marketing.attendance_settings` موجودًا في قاعدة البيانات القديمة كمكوّن Singleton بمفتاح `boolean` وقيد يضمن أن قيمة المفتاح `true`.

الإصدار السابق حاول تحويل المفتاح إلى `text` واستخدام القيمة `default`. عند وجود القيد القديم `id = true`، حاول PostgreSQL إعادة تفسير القيد بعد التحويل، فنتج الخطأ:

`operator does not exist: text = boolean`

## التنفيذ النهائي
- إلغاء تحويل نوع عمود `id` بالكامل.
- عدم مقارنة عمود `id` بأي قيمة داخل Runtime API.
- قراءة أحدث سجل إعدادات مباشرة باستخدام `updated_at`.
- تحديث سجل الإعدادات الوحيد باستخدام `ctid` داخليًا، بدون الاعتماد على نوع المفتاح.
- إنشاء الجدول في التثبيت الجديد بالنمط الأصلي المتوافق: مفتاح Singleton من نوع `boolean`.
- إدراج الإعدادات الافتراضية بدون تمرير قيمة لعمود `id`، بحيث يعمل مع الجدول القديم والحالي.
- تطبيق المنطق نفسه في Runtime Schema وملف Migration الأساسي.
