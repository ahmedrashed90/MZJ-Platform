# تنفيذ تكامل NEXT ERP مباشرة على سورس v1.19.7

تم تنفيذ التعديلات داخل المصدر الأساسي لنسخة `v1.19.7` دون استخدام أي ملف أو كود من `v1.19.8`، ودون إنشاء طبقة Patch جانبية.

## التعديلات المنفذة

- فصل التعرف على حدث الإلغاء عن جدول السيارات، بحيث يقبل Endpoint الموحد طلب `on_cancel` المختصر اعتمادًا على رقم الطلب و`creation` و`docstatus` حتى إذا لم يرسل NEXT ERP جدول `items`.
- الإبقاء على شرط جدول السيارات لحدث الاعتماد `on_submit`؛ فلا يُنشأ طلب تتبع ناقص بدون سيارات.
- توحيد هوية نسخة الطلب بين الاعتماد والإلغاء باستخدام `Sales Order No + creation`، مع استمرار السماح بإعادة استخدام نفس رقم الطلب لنسخة جديدة بعد الإلغاء.
- منع تحذيرات العميل والجوال والمستخدم غير اللازمة في Payload الإلغاء المختصر.
- إغلاق دورة الموافقات النشطة كسجل `cancelled` قبل الحفاظ على حالة السيارة إذا كانت `مباع تم التسليم` أو مؤرشفة؛ فلا تبقى دورة موافقات نشطة بعد إلغاء طلب NEXT ERP.
- لم يتم تغيير تصميم الصفحات أو الصلاحيات أو منطق CRM والتراكينج والعمليات خارج نطاق تكامل NEXT ERP.

## اختبارات الرجوع

أضيف الفحص:

```text
scripts/check-erpnext-clean-integration-v1197.mjs
```

ويغطي:

- اعتماد طلب متعدد السيارات.
- تجاهل صف رسوم التسجيل كسيارة.
- إلغاء مختصر بدون `items`.
- تطابق هوية النسخة بين الاعتماد والإلغاء.
- استمرار رفض `on_submit` بدون سيارات.
- ترتيب إغلاق دورة الموافقات قبل حفظ حالة السيارات المسلمة أو المؤرشفة.
