# نظام العمليات Native — الإصدار 1.14.0

تم بناء نظام العمليات داخل المنصة الموحدة اعتمادًا على React وTypeScript وAPI المنصة وPostgreSQL، مع استخدام تسجيل الدخول والمستخدمين والصلاحيات والقائمة الجانبية والتصميم الحالي للمنصة.

## الصفحات المضافة

- `/operations`: قاعدة بيانات السيارات والبحث والفلاتر والتصدير وفتح ملف السيارة.
- `/operations/manage`: إضافة وتعديل السيارات والاستيراد من XLSX أو CSV.
- `/operations/movements`: الحركة المباشرة الجماعية داخل Transaction واحدة.
- `/operations/requests`: طلبات النقل والتصوير ومراحلها الأربع.
- `/operations/availability`: تجميع السيارات حسب السيارة والبيان والموديل والألوان.
- `/operations/activity`: سجل الحركات والمستخدمين والتواريخ والتصدير.
- إعدادات العمليات داخل صفحة الإعدادات الموحدة.

## ما لم يتم نقله

- داش بورد العمليات القديمة.
- صفحة التراكينج القديمة أو أي iframe أو Firebase تابع لها.
- صفحة Media القديمة.
- إدارة مستخدمي النظام القديم.
- APIs واتساب وSMS القديمة الخاصة بالتراكينج.

## قاعدة البيانات والـAPI

تم الحفاظ على جداول `operations` الحالية التي تعتمد عليها الداش بورد الموحدة، مع توسعتها بدل استبدالها. أضيفت طبقة API مؤمنة على السيرفر، وصلاحيات `operations.*`، وسجل للأحداث، والتشييك، والملاحظات، والأرشفة المرنة، ودفعات الحركة.

تُطبّق توسعات قاعدة البيانات من خلال:

- `database/schema.sql`
- `server/_schema.ts`
- `server/_operations-schema.ts`

## قواعد التنفيذ المهمة

- لا يتم تنفيذ حركة جماعية إذا كان VIN واحد غير موجود أو غير صالح.
- نقل عدة سيارات يتم كله أو لا يتم شيء من خلال PostgreSQL Transaction.
- لا يمكن تحويل السيارة إلى «مباع تم التسليم» دون الموافقة المالية والإدارية.
- طلب النقل يغيّر مكان السيارة عند مرحلة «تم استلام السيارة» فقط.
- الأرشفة Soft Archive ولا تحذف السيارة أو سجلها.
- التحقق من ارتباط التراكينج يتم بواسطة VIN من جداول التراكينج الحالية، دون تكرار نظام التراكينج داخل العمليات.
- كل النوافذ الجديدة تغلق بزر Esc.
- الصلاحيات تُفحص في الـAPI، وليس بإخفاء الأزرار فقط.

## الفحوص المنفذة

- TypeScript build.
- Vite production build.
- فحص Imports الخاصة بالـAPI.
- جميع اختبارات Regression الحالية للـCRM والتراكينج والـWorkers.
- فحص مستقل لنظام العمليات للتأكد من Routes والصلاحيات والـTransactions وإغلاق النوافذ بزر Esc وعدم وجود Firebase أو iframe أو Media أو داش بورد قديمة داخل Module العمليات.

## ملاحظة النشر

تم التحقق من البناء والفحوص بصورة محلية. لم يتم تشغيل عمليات كتابة فعلية على قاعدة PostgreSQL الإنتاجية داخل بيئة العمل الحالية لعدم وجود اتصال إنتاجي، لذلك يجب نشر النسخة على بيئة اختبار متصلة بقاعدة البيانات وتشغيل اختبارات قبول ببيانات فعلية قبل تحويل المستخدمين إليها.
