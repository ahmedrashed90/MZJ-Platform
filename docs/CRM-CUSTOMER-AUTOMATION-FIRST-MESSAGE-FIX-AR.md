# إصلاح تشغيل الأوتوميشن من أول رسالة واردة

## المطلوب

أول رسالة واردة من عميل لا يوجد له فلو أوتوميشن نشط تبدأ الأوتوميشن دائمًا، بصرف النظر عن محتوى الرسالة.

## التنفيذ

تم تعديل محرك الأوتوميشن المركزي بحيث:

- أول رسالة واردة تنشئ تشغيلًا جديدًا بحالة البداية.
- يتم إرسال تسلسل البداية والترحيب وقائمة الخدمات وفق الإعدادات المحفوظة.
- لا يتم تفسير محتوى أول رسالة كاختيار خدمة.
- إذا كانت أول رسالة هي `1` أو `2` أو `3` أو اسم خدمة أو Payload زر، لا يتم تصنيف العميل مباشرة.
- اختيار الخدمة يبدأ فقط من الرسالة التالية بعد إرسال قائمة الخدمات.
- لم يتم تغيير منطق الفلو بعد اختيار الخدمة أو منطق التوزيع أو وركر فيسبوك.
- لا يحتاج التعديل إلى Migration أو تغيير قاعدة البيانات.

## الملفات المعدلة

- `scripts/check-crm-customer-automation-v1180.mjs`

- تمت إضافة فحوصات تمنع رجوع منطق اختيار الخدمة من أول رسالة مستقبلًا.
