# إصلاح صياغة SQL المركزي — v1.19.3

## سبب الخطأ

كانت قائمة `VALUES` الخاصة بجدول `core.system_pages` تنتهي بفاصلة بعد آخر صف مباشرة قبل `ON CONFLICT`، بالشكل غير الصحيح:

```sql
('tracking','delete','حذف طلبات التراكينج','/tracking/delete',30,true),
on conflict(system_code,code) ...
```

PostgreSQL ينتظر صفًا جديدًا بعد الفاصلة، لذلك يعيد `SQLSTATE 42601` عند الوصول إلى `ON`.

## التصحيح

تمت إزالة الفاصلة من آخر صف:

```sql
('tracking','delete','حذف طلبات التراكينج','/tracking/delete',30,true)
on conflict(system_code,code) ...
```

تم تطبيق التصحيح في المصادر الثلاثة التي تنفذ نفس الكتالوج:

1. Migration الصلاحيات المركزي.
2. Seed الكتالوج المركزي.
3. SQL التهيئة التلقائية داخل السيرفر.

لم تتم إضافة Migration ترقيعية أو منطق جانبي. تم تصحيح المصدر المركزي نفسه لأنه لم ينجح في التنفيذ من الأساس.

## إعادة التشغيل

بعد محاولة فاشلة داخل `BEGIN`، افتح Query جديدة أو نفذ:

```sql
rollback;
```

ثم شغّل ملف الـMigration المصحح كاملًا من بدايته. بعد نجاحه شغّل ملف Bootstrap الإصدار `1192`. ملف Seed مستقل وآمن للتكرار، لكنه ليس بديلًا عن الـMigration.
