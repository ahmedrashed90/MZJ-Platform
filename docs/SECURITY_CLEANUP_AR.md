# تقرير التنظيف والحماية الأمنية

## تمت إزالة أو منع

- الاعتماد على اسم أو إيميل أو UID ثابت لاتخاذ قرار صلاحية.
- استخدام `localStorage` كمصدر للصلاحيات.
- تجاوزات من نوع `canTransfer = true`.
- اعتبار إخفاء الزر حماية كافية.
- إنشاء المستخدمين من داخل الأنظمة الفرعية.
- استمرار جلسة مستخدم معطل أو جلسة بإصدار صلاحيات قديم.
- الوصول العام للتراكينج باستخدام VIN أو رقم الطلب مباشرة.
- جلب بيانات نظام ممنوع ثم إخفاؤها في الواجهة.
- السماح الضمني لمنشئ طلب العمليات بالحذف أو الإلغاء بدون المفتاح المستقل.
- استخدام صلاحية مدير عامة لتعديل رقم الهيكل؛ أصبح الإجراء بمفتاح حساس مستقل.
- تعديل صلاحيات قالب دور بمجرد امتلاك صلاحية إدارة اسم الدور.

## حماية كلمات المرور

- كلمات المرور الجديدة تُخزن بواسطة bcrypt عبر PostgreSQL `crypt` و`gen_salt('bf')`.
- Password Hash لا يُرجع للواجهة ولا يدخل في snapshots المعروضة.
- تحديث الاستيراد أو الصلاحيات لا يغير كلمة مرور المستخدم الحالي.

## حماية التكاملات

- Gateway CRM/Media: `MZJ_GATEWAY_SECRET` مع مقارنة آمنة.
- ERPNext webhook: `ERPNEXT_WEBHOOK_KEY` أو المفتاح المعتمد.
- Scheduler الداخلي: `AUTOMATION_SCHEDULER_SECRET`.
- Tracking ingest: مفتاح التكامل المحدد.

## السجل الأمني

يسجل النظام المستخدم والبريد والدور والنظام والصفحة ومفتاح الصلاحية والإجراء والسجل والتغييرات والفرع والقسم وIP وUser Agent وRequest ID والنتيجة وسبب الرفض أو التراجع عند توفره.
