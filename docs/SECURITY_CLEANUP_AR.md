# تقرير تنظيف الشروط الأمنية

## الشروط الثابتة

- لم يعد الوصول إلى إدارة المستخدمين يعتمد على كلمة `admin` داخل الصفحة أو API.
- لا يعتمد منطق الصلاحيات الجديد على اسم مستخدم أو بريد إلكتروني أو UID ثابت.
- الأدوار تستخدم كقوالب بيانات داخل PostgreSQL، وليست شرط سماح مباشر في الواجهة أو Endpoint.
- تمت إزالة مساعد مدير CRM الذي كان يحول أسماء أدوار محددة إلى سماح إداري.

## كلمات المرور

- إنشاء المستخدم يستخدم Hash من PostgreSQL عبر `crypt` و`gen_salt('bf')`.
- لا يعرض API كلمة المرور أو Password Hash.
- تحديث المستخدم لا يغير كلمة المرور المخزنة.
- المستخدم الجديد ملزم بتغيير كلمة المرور المؤقتة.

## الجلسات

- المستخدم المعطل يُرفض حتى لو كان لديه Token قديم.
- تعطيل المستخدم يحذف جلساته الحالية.
- تعديل الصلاحيات يرفع `permission_version` لإبطال النسخ القديمة من الصلاحيات.
