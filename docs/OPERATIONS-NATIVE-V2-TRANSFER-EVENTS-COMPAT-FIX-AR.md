# إصلاح توافق جدول أحداث طلبات النقل

## السبب الجذري
كانت قاعدة الإنتاج تحتوي مسبقًا على جدول `operations.transfer_request_events` بتركيب أقدم لا يتضمن العمود `transfer_request_id`.
أمر `CREATE TABLE IF NOT EXISTS` لم يعدّل الجدول الموجود، ثم حاول النظام إنشاء Index يعتمد على العمود غير الموجود، فظهر خطأ PostgreSQL `42703 / ComputeIndexAttrs`.

## الإصلاح
- إضافة العمود وكل أعمدة Native V2 الناقصة بصورة Idempotent قبل إنشاء الـIndex.
- محاولة ربط السجلات القديمة تلقائيًا إذا كان الجدول السابق يستخدم `request_id` أو `transfer_id`.
- عدم حذف أي سجل قديم.
- إبقاء الإصلاح داخل Runtime Schema، وإضافة Migration مستقلة قابلة للتشغيل يدويًا.

## بعد النشر
أول استدعاء لـ `/api/operations` يطبّق الإصلاح تلقائيًا، ثم يعيد تحميل بيانات `meta`.
