# مصفوفة إعادة بناء نظام التسويق Native

التاريخ: 2026-07-23

## المصدر والحدود

- المصدر الأساسي للتعديل: سورس منصة MZJ الموحدة المعتمد.
- سورس MZJ Marketing القديم استُخدم كمرجع وظيفي وتصميمي فقط.
- لم يتم نسخ `merged-patches.js` أو Firebase Runtime أو أي Monkey Patch.
- تم استبعاد **Checklist ريل السيارات** و**جدولة النشر المحلي** بناءً على طلب المستخدم.
- لا توجد صفحة إعدادات داخل التسويق؛ كل إعدادات التسويق موجودة داخل صفحة الإعدادات الموحدة.

## الفلو النهائي للحملة والمهام

- لا يوجد طلب هيكل مستقل.
- لكل Creative Instance × قسم تنفيذي × يوزر تنفيذي × كاتب محتوى يتم إنشاء Pair ثابت.
- لكل Pair مهمة `content_template` ومهمة `execution` تعتمد على Task ID الدقيقة.
- اعتماد Task Template يفتح مهمة التنفيذ المرتبطة بنفس Pair فقط.
- التقدم مصدره خدمة واحدة: إجراءات المهمة → متوسط القسم → متوسط الأقسام المتساوي للحملة.
- المهمة التي تتطلب ملفًا نهائيًا لا تصل إلى 100% بدون ملف صالح.
- الملفات ترفع إلى R2 عبر Presigned URL وتُربط بمسار المهمة الفعلي.

## الاستبعادات المقصودة

- لا Route ولا Navigation ولا Runtime لـChecklist ريل السيارات.
- لا Route ولا Agent ولا Runtime لجدولة النشر المحلي.
- لا صفحة إعدادات داخل التسويق.
- لا Firebase أو Firestore أو Firebase Storage.
- لا `merged-patches.js` ولا Wrapper أو Proxy أو MutationObserver.
